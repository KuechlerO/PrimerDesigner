Overview

PrimerDesigner is a web application that facilitates the design of primer pairs for variant-containing target regions in PCR and qPCR workflows. It supports different input types, including genomic positions, transcript-based variant descriptions, and complete sequences with annotated mutations.


Technical details

PrimerDesigner is built with modern web technologies and uses the following tools:

  • Django
  • Primer3-py
  • Dicey

For more information on the technical details, please refer to the README file.

Tutorial

Step-by-step guide

Before entering any data, the user can adjust several global settings in the web application. First, the reference genome can be selected as either GRCh37 or GRCh38. Second, the primer settings can either be left at the default configuration or adjusted manually (PCR and qPCR pre-sets are available). Third, the amplicon check can be enabled or disabled (disabled by default).

After configuring these general settings, one of three input modes can be used. The first mode is based on a genomic position. In this mode, a single nucleotide variant can be entered in genomic form by specifying the chromosome position together with the new base.

The second mode is based on a transcript ID. Here, the user can enter a transcript identifier and choose under Check against whether the variant should be interpreted relative to the genome or the transcript. In addition, the reference sequence type can be selected, for example cDNA or CDS. This mode supports both single nucleotide variants and indels. For substitutions, a variant position can be entered. For indels, the user can specify the wild-type start and end positions as well as the inserted bases.

The third mode is based on a full input sequence. In this mode, the user provides an entire sequence and receives primer suggestions directly for that sequence context. Sequence variants must be described explicitly within the input sequence. Substitutions are written in the format [A>T]. Indels can be represented with alternative notations such as [-/A], [T/-], or [ATG/CG], depending on the type of sequence change that should be described.

To explore example inputs for the three modes, the user can click the Load Example button repeatedly. Each click cycles through different example entries. In addition, every mode contains a small question mark icon in the lower right corner. When the user hovers over this icon, a more detailed explanation of the expected input format is displayed.

After clicking Generate Primers, the application displays the sequence context together with visual highlighting. Recommended primers are highlighted in turquoise blue, while the target region is marked in red. Further down the page, several candidate primers are listed for selection. For each candidate, additional properties such as penalty score, temperature, and GC content are shown.

The penalty score is especially important for interpretation: a lower penalty score indicates a better primer candidate. Once the user has selected a preferred primer pair, the Submit button can be used to continue to the next page. There, the user can click Download Report to obtain the final result as a report.
